Grammars

Grammars are used in two contexts:

  • They can be used by your speech-enabled application to specify what phrases to accept (For example, to accept move up three and not move up tree)
  • They can be used to enhance Voice Manager navigation of applications that are not speech-enabled. In addition to describing what phrases to accept, Voice Manager grammars also specify what action to take when a given phrase is recognized (for example, to send the keystrokes [up][up][up] when move up three is recognized.)

Basic steps to writing an application grammar

  • write a grammar file using a text editor. By convention, this file has the extension ".bnf".
  • compile the grammar file using VTBNFC.EXE By convention, the resulting file has the extension ".fsg".
  • write a file describing what will be displayed in the What Can I Say window. By convention, this file has the extension ".wcs"
  • combine the FSG and WCS files using TIDYGRAM.EXE (the resultant file has the extension ".fsg" by convention)
  • write a file that links a grammar to an application. This file is required to have the extension ".sgl"
